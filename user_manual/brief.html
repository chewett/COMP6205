Description of Web Application

We have decided to write a basic banking app which will allow a user to register, login and perform basic banking operations such as transferring money. Since there is a large amount of work involving in a banking app we will only be creating a subset of the pages and features that an actual website would have. Below is a list of the pages we plan to implement.

  • Log in
  • Register
  • Services
    • Transfer money
    • View statements
    • Open new account
    • View account
  • Contact us/About page
  • Admin page

In the technical documentation we will add comments describing other features we would add it to make it function like a real banking website. This is to demonstrate the web skills that we have improved by having a website that lets us do a number of interesting things.

For the role based authentication we have currently outlined 4 levels of access:

  • Customer
  • Clerk
  • Bank Manager
  • Admin

The roles will be organised in hierarchical manner, so that admins will be able to promote or demote people to be clerks or bank managers. Clerks will be able to view customer transactions and eventually resolve issues. Bank managers will be able to disable accounts and fire clerks

User Stories

We have identified parts of the user story that relate to features in our website and made them italic.

Customer - basic banking features
The customer will want to register an account. After that they can login and create bank account(s). Finally, they will be able to send/receive money.

Customer - print statements, report issues
After successful login, a customer will view and print their statement(s). In case there was a problem, they will be able to report the issue.

Bank Clerk
A clerk will want to add and remove money from a bank account. They will be using the interface in branch and as people present money to be cashed they will add this into the account.
Also, clerks will be able to view disputes or complaints, sent from customers. Customers will submit their issue and that will get saved to the database. Clerks, in turn, will fetch that information and once resolved, the issue will be marked as 'closed'.

Bank Manager
A bank manager will want to be able to do all that a clerk can do but also have the ability to disable accountsfor users that are not using their account properly and also register clerks.

Admin
An admin should be able to access the entire system and perform all actions that any other level of access can perform. Admins can also employ new managers and shut the site for regular maintenance.
