COMP6205 User Manual

Client Pages

  1. How to login

    • We have made the assumption that in order to get your login credentials, you must obtain them from the bank. Once you get your login details, you can go the login screen. Note that unless you are logged in, you cannot see anything but the Login and About us pages



    • The following screen shows the login screen where you have to put your username and password



    • If you cannot login or forget your password, you are encouraged to go to the About us page and see our contact details

  2. Account Overview

    • After a user logs in, they see the Account Overview page. Initially it would show no accounts and it would give the user the opportunity to open account or report issue.

  3. Open Account

    • Opening an account is as easy as giving a name of the account and specifying the type (such as savings, loan, etc)

    • After opening the new account, you are redirected to the Account Overview page where you can view your newly created account. Every new account has initial balance of 0£. (Mention that to get some money, you need to fiddle with DB or is that too technical?)

  4. Transfer money

    • After you have set up your accounts, you can finally transfer money between your accounts or even to other accounts, registered in the bank. To do the transfer (regardless whether it is between your accounts or to another person's account), you must know the id of the recipient. In the example below John Smith is sending 200£ from his daily account ( id: 1 ) to his savings ( id: 2 )


    • Note that when transfering money the bank website does NOT allow you to:
      • Send money to non-existing destination. This is why you MUST know the account id of the recipient,

      • Send more money than you have in the account from which you are sending.

      • Send negative amount of money.

      • Leave any of the fields blank.

  5. View Account Transactions and Statements

    • Immediately after you make a transaction, you are redirected to the Account Overview page where you can see the amount deducted from your account:

    • You can go into more detail and View Transactions for specific account, which will dynamically show you your most recent transactions for this account. (Button above was updated, so we probably need new screenshot). In the example below John Smith is observing the recent transactions for his savings account. You can see when the transaction took place, what is the reference and if money has come in or gone out of the account:

    • We realize that people have different lifestyles, so some may have 10 transactions for the last month or 100 for the past week. Therefore, apart from dynamically observing your recent transactions, you can also view your monthly statement. We see that John Smith has only one transaction in November:

    • You can navigate back and forth the months and see the statement for whichever month you are interested in. For instance, John Smith has gone back a month and is viewing his statement for November, when he had more activity on his account:

    • Finally, you can view a printable pdf of your montly statement and eventually download it:

  6. Report Concern

    7. From the Account Overview page you can submit an issue if you consider that there is something unsual going on. All you have to do is:

    • Select the account where the issue is present
    • Give a title to your problem
    • Describe the problem in more detail
    • After you have sent the report, a bank clerk will receive it and you will be contacted shortly.

  7. Log out

    8. Logging out simply logs you out of the bank website and redirects you to the Login page in case you want to login again

Admin pages

When an admin logs in, they can see the Admin tab in the navigation bar, which gives them more rights:


Recall that regular users do not see this page. Even if normal user guesses the URL for the admin page, they are not authorized to view it: ( this probably should not be in the user manual )


The admin page gives access to the following:

  1. Assign Users

    2. This allows the admin to change what roles a user has. E.g. promoting a user from a normal user to a bank clerk, promoting clerk to manager or demoting manager to clerk. However, as this is not a full website, so this feauture is not implemented: Again, is this suitable for the user manual?


  2. View Roles

    3. This page shows the admin the list of available roles and a short description of the role:


  3. Change Permissions

    4. This allows the admin to change the permissions of different roles, e.g what capabilities would the user, the clerk, the manager and the admin have. However, as this is not a full website, so this feauture is not implemented: Again, is this suitable for the user manual?


  4. View Issues

    5. This page allows the admin, Joe Bloggs, to view issues, submitted by clients


    After choosing an issue, submitted by particular client, the admin, Joe Bloggs, can view the case in more detail


    After contacting the client, the admin, Joe Bloggs, can close the issue, after which he gets a confirmation


    Finally, when going back to the issues list, the admin, Joe Bloggs, can see that the issues that he has resolved are now under the 'Resolved' category


  5. View Site Options

    6. Finally, the admin is allowed to view site options. For example, he would be able to temporarily bring down the site for regular maintenance. However, as this is demo site, this feauture is not implemented.


  6. Log out

    7. Logging out simply logs you out of the bank website and redirects you to the Login page in case you want to login again
