COMP6205 User Manual

Manual Usage

When using this user manual and website we recommend you use an updated browser of your choice. Older browsers will work, but will have reduced functionality and some features may not be available.

The website and manual will work on tablets and mobile devices but again for the best user experience you should view it on a desktop computer and browser.

Client Pages

These help topics relate to ones that a user may access when using our website.

How to login

We have made the assumption that you must obtain your login details from the bank. Once you get your details you will be able to login. Note that unless you are logged in, you cannot see anything but the Login and About us pages links at the top of the page.

The following screen shows the login screen to insert your username and password

If you cannot login or have forgotten your password, you are encouraged to go to the About us page and see our contact details

Account Overview

After a user logs in, they see the Account Overview page. Initially it will show no accounts and will give the user the opportunity to open an account or report an issue.

Open Account

Opening an account is easy and you can do this by giving a name for the account and specifying the type (such as savings, loan, etc)

After opening the new account, you are redirected to the Account Overview page where you can view your newly created account. Every new account has initial balance of £0

Transfer money

After you have set up your accounts, you can finally transfer money between your accounts or even to other accounts, registered in the bank. To perform this transfer (regardless whether it is between your accounts or to another person's account), you must know the id of the recipient's account (This is similar to the sort code/bank account ID in a real bank). In the example below John Smith is sending £200 from his daily account ( id: 1 ) to his savings ( id: 2 )

Note that when transferring money the bank website does not allow you to:

  • Send money to non-existing destination. This is why you must know the account id of the recipient,



  • Send more money than you have in the account you are sending from.



  • Send negative amounts of money.



  • Leave any of the fields blank.



View Account Transactions and Statements

Immediately after you make a transaction, you are redirected to the Account Overview page where you can see the amount deducted from your account.

You can go into more detail and View Transactions for specific account, which will dynamically show you your most recent transactions for this account. In the example below John Smith is observing the recent transactions for his savings account. Report Concern

From the Account Overview page you can submit an issue if you consider that there is something unusual going on. To do this you need to:

  • Select the account where the issue is present
  • Give a title to your problem
  • Describe the problem in more detail
  • After you have sent the report, a bank clerk will receive it and you will be contacted shortly.

Log out

To log out you can press the Log Out button at the top of the page. This will log you out and redirect you to the main login page if you wish to login later.

Admin pages

These help topics relate to ones that an admin may access when using our website.

When an admin logs in, they can see the Admin tab in the navigation bar, which gives them more rights:


Recall that regular users do not see this page. Even if normal user guesses the URL for the admin page, they are not authorised to view it: ( this probably should not be in the user manual )


The admin page gives access to the following:

Assign Users

This allows the admin to change what roles a user has. E.g. promoting a user from a normal user to a bank clerk, promoting clerk to manager or demoting manager to clerk. However, as this is not a full website, this feature is not implemented.


View Roles

This page shows the admin the list of available roles and a short description of the role:


Change Permissions

This allows the admin to change the permissions of different roles, e.g what capabilities would the user, the clerk, the manager and the admin have. However, as this is not a full website, this feature is not implemented hower this is only a matter of changing fields in the database.


View Issues

This page allows the admin, Joe Bloggs, to view issues, submitted by clients


After choosing an issue, submitted by particular client, the admin, Joe Bloggs, can view the case in more detail


After contacting the client, the admin, Joe Bloggs, can close the issue, after which he gets a confirmation


Finally, when going back to the issues list, the admin, Joe Bloggs, can see that the issues that he has resolved are now under the 'Resolved' category


View Site Options

Finally, the admin is allowed to view site options. For example, he would be able to temporarily bring down the site for regular maintenance. However, as this is demo site, this feature is not implemented.


User Stories

From the user brief we have refined and combined our user stories to relate to individual features for the website. We will link to the above guides to show that these features are available to complete their user story. This has formed part of our user acceptance testing.

Customer

The customer will want to use the all basic functionality of the banking application. From the brief both Customers have been combined into one user story.

  • Able to log into the website - The above instruction demonstrates how a user can log into the system with their bank provided credentials
  • Create a bank account - The above instructions demonstrates a user can create a bank account.
  • Send and receive money - The above instructions demonstrates a user can send and receive money.
  • View statements - The above instruction demonstrates how a user can view each months statement.
  • Print statements - The above instructions demonstrates how a user can print off their statement for review.
  • Report issues - The above instruction demonstrates how a user can report an issue and tells them how it is resolved.

Bank Manager

The Bank Manger will want the ability to use the application to monitor and resolve issues. From the brief both Bank Manager and have been combined into one user story.

  • Review and close issues - The above instruction demonstrates how a user can review and close the issues. They are expected to have resolved them by contacting the customer before marking them as resolved.
  • Alter/disable accounts - The above instruction demonstrates how a user can alter accounts.
  • Change User Roles - The above instruction demonstrates how a user can also change what role a user can have.

Admin

The admin will want the ability to perform administrative tasks relating to the site such as shutting it down when needed.

  • Perform all previous users can - This is demonstrated above by showing system features
  • Change site options - The above instruction demonstrates how an admin can change site options. Specifically they will be able to turn on maintenance mode.
