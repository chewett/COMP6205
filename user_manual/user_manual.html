COMP6205 User Manual

Manual Usage

When using this user manual and website we recommend you use an updated browser of your choice. Older browsers will work, but will have reduced functionality and some features may not be available.

The website and manual will work on tablets and mobile devices but again for the best user experiance you should view it on a desktop computer and browser.

Client Pages

These help topics relate to ones that a user may access when using our website.

How to login

We have made the assumption that in order to get your login credentials, you must obtain them from the bank. Once you get your login details, you can go the login screen. Note that unless you are logged in, you cannot see anything but the Login and About us pages

Account Overview

After a user logs in, they see the Account Overview page. Initially it will show no accounts and will give the user the opportunity to open an account or report an issue.

Open Account

Opening an account is as easy as giving a name of the account and specifying the type (such as savings, loan, etc)

After opening the new account, you are redirected to the Account Overview page where you can view your newly created account. Every new account has initial balance of 0£. (Mention that to get some money, you need to fiddle with DB or is that too technical?)

Transfer money

After you have set up your accounts, you can finally transfer money between your accounts or even to other accounts, registered in the bank. To do the transfer (regardless whether it is between your accounts or to another person's account), you must know the id of the recipient. In the example below John Smith is sending 200£ from his daily account ( id: 1 ) to his savings ( id: 2 )

Note that when transferring money the bank website does not allow you to:

  • Send money to non-existing destination. This is why you must know the account id of the recipient,



  • Send more money than you have in the account from which you are sending.



  • Send negative amount of money.



  • Leave any of the fields blank.

Report Concern

From the Account Overview page you can submit an issue if you consider that there is something unsual going on. To do this you need to:

  • Select the account where the issue is present
  • Give a title to your problem
  • Describe the problem in more detail
  • After you have sent the report, a bank clerk will receive it and you will be contacted shortly.

Log out

To log out you can press the Log Out button at the top of the page. This will log you out and redirect you to the main login page if you wish to login later.

When an admin logs in, they can see the Admin tab in the navigation bar, which gives them more rights:

Assign Users

This allows the admin to change what roles a user has. E.g. promoting a user from a normal user to a bank clerk, promoting clerk to manager or demoting manager to clerk. However, as this is not a full website, this feature is not implemented.


View Roles

This page shows the admin the list of available roles and a short description of the role:


Change Permissions

This allows the admin to change the permissions of different roles, e.g what capabilities would the user, the clerk, the manager and the admin have. However, as this is not a full website, this feature is not implemented.


View Issues

This page allows the admin, Joe Bloggs, to view issues, submitted by clients


After choosing an issue, submitted by particular client, the admin, Joe Bloggs, can view the case in more detail


After contacting the client, the admin, Joe Bloggs, can close the issue, after which he gets a confirmation


Finally, when going back to the issues list, the admin, Joe Bloggs, can see that the issues that he has resolved are now under the 'Resolved' category


View Site Options

Finally, the admin is allowed to view site options. For example, he would be able to temporarily bring down the site for regular maintenance. However, as this is demo site, this feauture is not implemented.


User Stories

We have identified parts of the user story that relate to features in our website and made them italic.

Customer - basic banking features
The customer will want to register an account. After that they can login and create bank account(s). Finally, they will be able to send/receive money.

Customer - print statements, report issues
After successful login, a customer will view and print their statement(s). In case there was a problem, they will be able to report the issue.

Bank Clerk
A clerk will want to add and remove money from a bank account. They will be using the interface in branch and as people present money to be cashed they will add this into the account.
Also, clerks will be able to view disputes or complaints, sent from customers. Customers will submit their issue and that will get saved to the database. Clerks, in turn, will fetch that information and once resolved, the issue will be marked as 'closed'.

Bank Manager
A bank manager will want to be able to do all that a clerk can do but also have the ability to disable accountsfor users that are not using their account properly and also register clerks.

Admin
An admin should be able to access the entire system and perform all actions that any other level of access can perform. Admins can also employ new managers and shut the site for regular maintenance.
