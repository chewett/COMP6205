Description of Web Application

The app will have basic banking features:

  • Log in
  • Register
  • Services
    • Transfer money
    • View statements
    • Open new account
    • View account
  • Contact us/About page
  • Admin page

There will also be different roles such as:

  • Customer
  • Clerk
  • Bank Manager
  • Admin

The roles will be organised in hierarchical manner, so that admins will be able to promote or demote people to be clerks or bank managers. Clerks will be able to view customer transactions and eventually resolve issues. Bank managers will be able to disable accounts and fire clerks

User Stories

      A user will want to create an account. Once created they should be able to login and create an account. For the purposes of this prototype accounts are created automatically and need no further approval. If we were to continue working with this we will ensure that bank managers approve account creation.
      A clerk will want to add and remove money from an account. They will be using the interface in branch and as people present money to be cashed they will add this into the account. In a real situation this would be patrolled and properly dealt with but we are not concerning ourselves with this.
      A bank manager will want to be able to do all that a Clerk can do but also have the ability to disable accounts for users that are not using their account properly and also register clerks.
      An admin should be able to access the entire system and perform all actions that any other level of access can perform.
